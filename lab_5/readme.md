# Лабораторная работа №5. Объектно-ориентированное программирование в PHP

## Условие на Github

Лабораторная работа посвящена разработке PHP-приложения для управления банковскими транзакциями с использованием объектно-ориентированного подхода.

## Цель работы

Освоить основы объектно-ориентированного программирования в PHP на практике. Научиться создавать собственные классы, использовать инкапсуляцию для защиты данных, разделять ответственность между классами, а также применять интерфейсы для построения гибкой архитектуры приложения.

## Условие

Необходимо разработать приложение для управления банковскими транзакциями.

Приложение должно позволять:

- хранить банковские транзакции;
- добавлять новые транзакции;
- удалять транзакции;
- искать транзакции;
- сортировать транзакции;
- выполнять вычисления над коллекцией транзакций;
- выводить данные в виде HTML-таблицы.

В рамках лабораторной работы необходимо использовать объектно-ориентированный подход.

Все классы можно разместить:

- либо в одном файле `index.php` для упрощения;
- либо в отдельных файлах, если требуется более аккуратная структура проекта.

## Структура проекта

В лабораторной работе использованы отдельные файлы для каждого компонента:

- `Transaction.php` — класс одной банковской транзакции;
- `TransactionStorageInterface.php` — интерфейс хранилища транзакций;
- `TransactionRepository.php` — репозиторий для хранения и поиска транзакций;
- `TransactionManager.php` — бизнес-логика для вычислений и сортировки;
- `TransactionTableRenderer.php` — генерация HTML-таблицы;
- `getMockTransactions.php` — начальные тестовые данные;
- `index.php` — точка входа и вывод готовой страницы.

## Отчет по проделанной работе

### Задание 1. Включение строгой типизации

Во всех основных файлах лабораторной работы включена строгая типизация:

```php
<?php

declare(strict_types=1);
```

Это позволяет уменьшить количество ошибок, связанных с неявным приведением типов, и делает код более предсказуемым.

### Задание 2. Класс `Transaction`

Создан класс `Transaction`, описывающий одну банковскую транзакцию.

Реализованы приватные свойства:

- `id` — уникальный идентификатор;
- `date` — дата транзакции;
- `amount` — сумма;
- `description` — описание платежа;
- `merchant` — получатель платежа.

Все значения передаются через конструктор. Для доступа к данным реализованы getter-методы. Для работы с датой используется класс `DateTime`.

Также добавлен метод:

- `getDaysSinceTransaction(): int` — вычисляет количество дней между датой транзакции и текущей датой.

### Задание 3. Класс `TransactionRepository`

Создан класс `TransactionRepository`, который хранит коллекцию объектов `Transaction` в приватном массиве и предоставляет базовые методы доступа:

- `addTransaction(Transaction $transaction): void`
- `removeTransactionById(int $id): void`
- `getAllTransactions(): array`
- `findById(int $id): ?Transaction`

Таким образом, доступ к данным осуществляется только через методы класса, без прямой работы с массивом транзакций извне.

### Задание 4. Класс `TransactionManager`

Создан класс `TransactionManager`, отвечающий за бизнес-логику приложения. Он не хранит транзакции самостоятельно, а получает доступ к ним через объект хранилища, переданный в конструктор.

Реализованы методы:

- `calculateTotalAmount(): float` — вычисление общей суммы всех транзакций;
- `calculateTotalAmountByDateRange(string $startDate, string $endDate): float` — вычисление суммы транзакций за указанный период;
- `countTransactionsByMerchant(string $merchant): int` — подсчет количества транзакций по получателю;
- `sortTransactionsByDate(): array` — сортировка транзакций по дате;
- `sortTransactionsByAmountDesc(): array` — сортировка транзакций по сумме по убыванию.

Для работы используются `array_reduce`, `array_filter`, `usort` и объекты `DateTime`.

### Задание 5. Класс `TransactionTableRenderer`

Создан отдельный класс `TransactionTableRenderer`, который отвечает только за вывод данных в HTML.

Реализован метод:

- `render(array $transactions): string`

Метод формирует HTML-таблицу и возвращает ее в виде строки. В основном файле остается только вызов метода и вывод результата через `echo`.

В таблице выводятся:

- ID транзакции;
- дата;
- сумма;
- описание;
- получатель;
- количество дней с момента транзакции.

Класс объявлен как `final`, что соответствует рекомендации задания.

### Задание 6. Начальные данные

Создано 10 объектов `Transaction` с различными:

- датами;
- суммами;
- описаниями;
- получателями.

Начальные данные вынесены в файл `getMockTransactions.php`, после чего транзакции добавляются в `TransactionRepository`.

### Задание 7. Интерфейс `TransactionStorageInterface`

Для повышения гибкости архитектуры создан интерфейс `TransactionStorageInterface`.

Интерфейс содержит методы:

- `addTransaction(Transaction $transaction): void`
- `removeTransactionById(int $id): void`
- `getAllTransactions(): array`
- `findById(int $id): ?Transaction`

После этого:

- `TransactionRepository` реализует интерфейс `TransactionStorageInterface`;
- `TransactionManager` зависит не от конкретного класса, а от интерфейса.

Конструктор `TransactionManager` приведен к следующему виду:

```php
public function __construct(
    private TransactionStorageInterface $repository
) {
}
```

## Контрольные вопросы

### 1. Зачем нужна строгая типизация в PHP и как она помогает при разработке?

Строгая типизация позволяет PHP проверять соответствие типов аргументов и возвращаемых значений без неявного преобразования. Это помогает раньше обнаруживать ошибки, упрощает отладку и делает поведение программы более предсказуемым.

### 2. Что такое класс в объектно-ориентированном программировании и какие основные компоненты класса вы знаете?

Класс — это шаблон для создания объектов. Он описывает состояние и поведение будущих экземпляров. Основные компоненты класса:

- свойства;
- методы;
- конструктор;
- модификаторы доступа (`public`, `private`, `protected`);
- при необходимости константы и статические элементы.

### 3. Объясните, что такое полиморфизм и как он может быть реализован в PHP

Полиморфизм — это возможность работать с объектами разных классов через общий интерфейс или общий базовый тип. В PHP он реализуется через интерфейсы и наследование. Например, если несколько классов реализуют один интерфейс хранения транзакций, `TransactionManager` сможет работать с любым из них одинаково.

### 4. Что такое интерфейс в PHP и как он отличается от абстрактного класса?

Интерфейс определяет набор методов, которые класс обязан реализовать, но не содержит состояния объекта. Абстрактный класс может содержать как абстрактные, так и обычные методы, а также свойства. Интерфейс задает контракт, а абстрактный класс чаще используется для частичной общей реализации.

### 5. Какие преимущества дает использование интерфейсов при проектировании архитектуры приложения? Объясните на примере данной лабораторной работы

Интерфейсы уменьшают связанность компонентов и делают архитектуру гибче. В данной лабораторной работе `TransactionManager` зависит от `TransactionStorageInterface`, а не от `TransactionRepository`. Это значит, что в будущем можно заменить текущее хранилище, например, на файловое или базу данных, не изменяя код менеджера, если новый класс будет реализовывать тот же интерфейс.
