# Лабораторная работа №6. Обработка и валидация форм

## Тема проекта

В качестве темы выбрано приложение **«Дневник настроения»**. Пользователь может заполнить форму, отправить данные на сервер, пройти валидацию и сохранить запись в JSON-файл. После этого записи выводятся в HTML-таблице с возможностью сортировки.

## Модель данных

Для одной записи используются следующие поля:

1. `title` — заголовок записи, `string`;
2. `author` — имя автора, `string`;
3. `entry_date` — дата записи, `date`;
4. `mood` — настроение, `enum`;
5. `energy_level` — уровень энергии, `enum`;
6. `sleep_hours` — количество часов сна, `float`;
7. `activities` — выбранные активности, `checkbox[]`;
8. `gratitude` — короткий текст благодарности, `text`;
9. `notes` — подробные заметки о дне, `text`;
10. `created_at` — дата и время создания записи, `date`.

Таким образом, задание покрывает все обязательные требования:

- больше 6 полей;
- есть строковые поля;
- есть поля даты;
- есть ограниченный набор значений;
- есть длинные текстовые поля.

## Архитектура решения

Решение выполнено в ООП-стиле. В папке `lab_6` используются следующие классы:

- `MoodEntry` — сущность одной записи дневника;
- `MoodEntryForm` — нормализация и серверная валидация формы;
- `ValidatorInterface` — общий интерфейс валидаторов;
- `RequiredValidator`, `StringLengthValidator`, `DateValidator`, `ChoiceValidator`, `NumberRangeValidator`, `ArrayChoiceValidator` — отдельные валидаторы;
- `MoodEntryStorageInterface` — интерфейс хранилища;
- `JsonMoodEntryRepository` — сохранение и чтение данных из `data/mood_entries.json`;
- `MoodEntryManager` — бизнес-логика создания и сортировки записей;
- `MoodEntryTableRenderer` — генерация HTML-таблицы;
- `Html` — экранирование пользовательских данных при выводе.

Дополнительно реализован интерфейс валидаторов, поэтому работа закрывает и пункт с расширением архитектуры через интерфейс.

## Что реализовано по шагам задания

### Шаг 1. Модель данных

Спроектирована модель данных для дневника настроения, содержащая 10 полей.

### Шаг 2. HTML-форма

В `index.php` создана HTML-форма с методом `POST`. Для улучшения пользовательского опыта добавлены клиентские ограничения:

- `required`;
- `minlength`;
- `maxlength`;
- `min`, `max`, `step`;
- `type="date"`;
- `type="number"`;
- checkbox-группа `activities[]`.

### Шаг 3. Обработка данных на сервере

После отправки формы:

1. данные читаются из `$_POST`;
2. создается объект `MoodEntryForm`;
3. запускается серверная валидация;
4. при успехе создается объект `MoodEntry`;
5. запись сохраняется в `data/mood_entries.json`;
6. пользователю выводится сообщение об успешном сохранении или ошибках.

### Шаг 4. Вывод данных

Все записи читаются из JSON-файла и выводятся в HTML-таблице. Реализована сортировка:

- по дате создания;
- по дате записи;
- по настроению;
- по автору;
- по количеству часов сна.

Сортировку можно менять через `GET`-параметры или через клики по заголовкам таблицы.

### Шаг 5. Дополнительная функция

Для повышения оценки реализован интерфейс `ValidatorInterface`, который унифицирует работу всех валидаторов. Это делает код расширяемым: новый валидатор можно добавить без изменения общей схемы обработки формы.

## Формат хранения данных

Данные сохраняются в файл:

- `lab_6/data/mood_entries.json`

Формат хранения — JSON с читаемым отступом (`JSON_PRETTY_PRINT`), поэтому записи легко просматривать вручную.

## Безопасность

Для защиты от XSS все пользовательские данные при выводе проходят через `htmlspecialchars()` внутри класса `Html`.

## Контрольные вопросы

### 1. Какие существуют методы отправки данных из формы на сервер? Какие методы поддерживает HTML-форма?

Основные методы отправки данных — `GET` и `POST`. Стандартная HTML-форма напрямую поддерживает именно эти два метода через атрибут `method`.

### 2. Какие глобальные переменные используются для доступа к данным формы в PHP?

Чаще всего используются:

- `$_GET` — данные, отправленные методом `GET`;
- `$_POST` — данные, отправленные методом `POST`;
- `$_REQUEST` — объединенный массив данных запроса.

Дополнительно могут использоваться `$_FILES` для загруженных файлов и `$_SERVER` для информации о запросе.

### 3. Как обеспечить безопасность при обработке данных из формы, например защититься от XSS?

Нужно:

- валидировать входные данные на сервере;
- не доверять данным из формы;
- экранировать вывод через `htmlspecialchars()`;
- ограничивать допустимые значения для select/checkbox/radio;
- проверять типы, длину строк и формат даты;
- по возможности использовать шаблон PRG, чтобы избежать повторной отправки формы.
