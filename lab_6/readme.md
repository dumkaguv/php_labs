# Лабораторная работа №6

## Тема проекта
Дневник настроения.

## Цель работы
Освоить обработку HTML-форм в PHP, включая отправку данных на сервер, валидацию, сохранение данных в файл и вывод записей в виде таблицы.

## Что реализовано
- HTML-форма для добавления новой записи дневника настроения
- отправка данных методом `POST`
- клиентская валидация через `required`, `minlength`, `maxlength`, `min`, `max`
- серверная валидация данных
- сохранение записей в файл `JSON`
- вывод записей в HTML-таблице
- сортировка записей по нескольким полям
- простая объектно-ориентированная реализация
- PHPDoc-комментарии для классов и методов

## Модель данных
В проекте используется сущность записи дневника настроения со следующими полями:

1. `id` - идентификатор записи
2. `title` - заголовок записи, тип `string`
3. `author` - автор записи, тип `string`
4. `entry_date` - дата записи, тип `date`
5. `mood` - настроение, тип `enum`
6. `activities` - список активностей, тип `checkbox`
7. `notes` - заметки, тип `text`
8. `created_at` - дата и время создания записи
9. `sleep_hours` - количество часов сна, тип `int`

Таким образом, в работе есть:
- больше 6 полей
- поле типа `string`
- поле типа `date`
- поле типа `enum`
- поле типа `text`

## Краткое описание работы программы
1. Пользователь открывает страницу с формой.
2. Заполняет поля и отправляет форму методом `POST`.
3. Данные попадают в `$_POST`.
4. Класс `MoodValidator` проверяет корректность введенных значений.
5. Если есть ошибки, они показываются пользователю.
6. Если ошибок нет, создается объект `MoodRecord`.
7. Класс `MoodRepository` сохраняет запись в файл `JSON`.
8. После этого все записи считываются из файла и выводятся в таблице.
9. Пользователь может сортировать таблицу по автору, дате записи, настроению и дате создания.

## Почему использовано ООП
Для получения 10 баллов была выбрана объектно-ориентированная реализация.

В работе классы разделены по ответственности:
- `MoodRecord` хранит данные одной записи
- `MoodValidator` отвечает за проверку данных
- `MoodRepository` отвечает за сохранение и чтение данных
- `MoodApp` координирует работу приложения

Такой подход делает код более понятным, удобным для поддержки и расширения.

## Валидация данных
### Клиентская валидация
На стороне браузера используются:
- `required`
- `minlength`
- `maxlength`
- `min`
- `max`

### Серверная валидация
На стороне сервера проверяется:
- заполнен ли заголовок
- заполнен ли автор
- корректна ли дата
- выбрано ли настроение
- выбрана ли хотя бы одна активность
- заполнены ли заметки
- входит ли количество часов сна в допустимый диапазон

## Хранение данных
Данные сохраняются в файл `data/entries.json`.

Формат хранения выбран `JSON`, потому что:
- он читаемый
- его удобно проверять вручную
- его легко считывать и записывать в PHP

## Сортировка
Сортировка реализована через `GET`-параметры:
- `sort`
- `direction`

Можно сортировать записи по:
- автору
- дате записи
- настроению
- дате создания

## Безопасность
Для защиты от XSS данные перед выводом экранируются через `htmlspecialchars()` в классе `Html`.

Это нужно, чтобы пользовательский ввод не выполнялся как HTML или JavaScript-код.

## Контрольные вопросы
### Какие существуют методы отправки данных из формы на сервер? Какие методы поддерживает HTML-форма?
Основные методы HTML-формы - это `GET` и `POST`.

- `GET` передает данные через URL
- `POST` передает данные в теле запроса

В данной работе для отправки формы используется `POST`, а для сортировки таблицы используется `GET`.

### Какие глобальные переменные используются для доступа к данным формы в PHP?
Основные суперглобальные массивы:
- `$_GET`
- `$_POST`
- `$_REQUEST`
- `$_FILES`
- `$_SERVER`

В этой работе используются:
- `$_POST` для данных формы
- `$_GET` для параметров сортировки
- `$_SERVER` для проверки метода запроса

### Как обеспечить безопасность при обработке данных из формы, например защититься от XSS?
Для защиты от XSS нужно:
- не выводить пользовательские данные напрямую
- экранировать вывод через `htmlspecialchars()`
- выполнять серверную валидацию данных
- не доверять данным, пришедшим из браузера

В данной работе защита от XSS реализована через класс `Html`, который использует `htmlspecialchars()`.
